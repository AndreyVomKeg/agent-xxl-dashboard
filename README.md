# Agent XXL — Мультиагентная система управления Google Ads

## Обзор

Agent XXL — AI-платформа для автоматизации Google Ads на базе четырёх специализированных агентов. Каждый агент отвечает за свою область: аналитика, тексты, ставки, аудит. Система включает загрузку пользовательских скиллов (знаний) с полным пайплайном валидации.

## Текущий статус

**Frontend-прототип** — интерактивный React-дашборд (single-file JSX) с имитацией данных и полным UI для всех функций.

Файл: `google-ads-ai-dashboard.jsx`

---

## Архитектура агентов

### 1. Аналитик
- **Роль:** мониторинг метрик, обнаружение аномалий, прогнозирование
- **Цвет:** синий (#4285F4)
- **Инструменты:** Google Ads API (отчёты), статистические модели, алерты
- **Пример:** «СTR кампании "Конкуренты" упал на 32% за 3 дня»

### 2. Копирайтер
- **Роль:** генерация текстов объявлений, A/B тестирование, анализ посадочных
- **Цвет:** жёлтый (#D09D00)
- **Инструменты:** LLM-генерация, шаблоны форматов, анализ конкурентов
- **Пример:** «3 новых варианта заголовков для кампании "Услуги"»

### 3. Оптимизатор
- **Роль:** управление ставками, перераспределение бюджетов, расписание показов
- **Цвет:** зелёный (#34A853)
- **Инструменты:** bidding API, бюджетные правила, гео-аналитика
- **Пример:** «Ретаргетинг ROAS 5.1 → увеличить бюджет на 25%»

### 4. Аудитор
- **Роль:** проверка качества аккаунта, конфликты минус-слов, Quality Score
- **Цвет:** красный (#EA4335)
- **Инструменты:** чеклисты, правила QS, валидация структуры
- **Пример:** «Минус-слово "ремонт" блокирует целевой запрос»

## Структура дашборда

### Вкладка «Обзор»
- 4 метрик-карты: клики, показы, конверсии, CPA (с трендами)
- Area-график динамики (клики + конверсии, двойная ось Y)
- Pie-чарт каналов трафика (Поиск, РСЯ, Ретаргетинг, Видео)
- Таблица кампаний: 6 колонок (название, статус, бюджет/расход, CPA, ROAS, тренд)

### Вкладка «Кампании»
- Раскрываемые карточки каждой кампании
- Внутри: ключевые слова (с QS и CPA), объявления (с CTR и статусом A/B)

### Вкладка «Агенты»
- 4 summary-карты агентов (иконка, статус, impact, кол-во скиллов)
- Bar-чарт эффективности (задачи vs принятые рекомендации)
- Раскрываемые карточки агентов с 3 подвкладками:
  - **Скиллы** — текущая задача, действия, очередь + загрузка файлов + список скиллов
  - **Версии** — таймлайн версий с метриками, кнопка отката
  - **Метрики влияния** — до/после + линейный график сравнения

### Правая панель (AI)
- Раскрываемый блок рекомендаций (4 алерта от агентов с кнопками действий)
- Чат с ассистентом (поддерживает запросы о CPA, бюджете, объявлениях, скиллах)

## Skill Pipeline

Ключевая фича — загрузка пользовательских знаний для агентов через UI с интеллектуальной обработкой.

### Пользовательский сценарий (реализован в UI)

1. **Загрузка** — drag & drop или клик, форматы PDF/MD/TXT/JSON
2. **Обработка** — модальное окно с прогресс-баром (парсинг → извлечение → LLM-анализ → экстракция → проверка)
3. **Ревью правил** — список извлечённых правил с чекбоксами, inline-редактированием, удалением. Каждое правило имеет тип (tone, restrict, format, audience, usp) и приоритет (high/medium/low)
4. **Разрешение конфликтов** — split-view (новое vs существующее), 3 варианта: применить новое, оставить старое, оставить оба
5. **Применение** — зелёный чекмарк, обновление версии, уведомление о метриках через 7 дней

### Архитектура бэкенда (спроектирована, не реализована)

**Слой 1 — Приём и парсинг:**
- PDF → OCR + текст + таблицы; MD/TXT → парсинг; JSON → валидация
- Отсев: битые файлы, дубликаты, нераспознаваемые языки
- Хранение оригинала в S3, метаданные в PostgreSQL

**Слой 2 — Интеллектуальная экстракция:**
- LLM извлекает типизированные объекты (правила тона, запреты, пороги, чеклисты)
- Каждый объект привязан к фрагменту исходного документа
- Пользователь подтверждает перед применением

**Слой 3 — Конфликт-менеджер и RAG:**
- Семантическое сравнение эмбеддингов новых и существующих правил
- Векторная индексация в Qdrant/pgvector
- При вызове агента: top-K релевантных чанков → контекст LLM

**Слой 4 — Версионирование и метрики:**
- Снимки версий (файлы, правила, настройки)
- Метрики до/после (CTR, CPA, ROAS, QS) через 7 дней
- A/B тестирование наборов скиллов

### Планируемый стек
- PostgreSQL — метаданные, версии, правила
- Qdrant / pgvector — векторный поиск
- Redis — кеширование скиллов
- Celery / Temporal — асинхронная обработка
- LangChain — retriever для подстановки скиллов
- WebSocket — реалтайм-статусы обработки

## Распределение Frontend / Backend

| Зона | Frontend | Backend |
|------|----------|---------|
| Валидация файлов | Формат, размер, кодировка | OCR, парсинг, экстракция |
| Обработка | Прогресс-бар, статусы (WS) | LLM-вызовы, индексация |
| Правила | Inline-edit, чекбоксы | Семантические конфликты |
| Версии | Таймлайн, сравнение | Снимки, хранение |
| Метрики | Графики до/после | Сбор данных, A/B тесты |

Соотношение по сложности: ~60% бэкенд / ~40% фронтенд.
