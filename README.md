# Agent XXL — Мультиагентная AI-платформа для Google Ads

> Управление рекламой Google Ads на базе четырёх специализированных агентов

## Проблема

Управление рекламой в Google Ads требует постоянного внимания: мониторинг метрик, оптимизация ставок, генерация текстов, проверка качества аккаунта. Даже опытный специалист не может отслеживать всё одновременно — **аномалии обнаруживаются поздно, бюджет тратится неэффективно, конфликты между кампаниями остаются незамеченными.**

Существующие инструменты автоматизации решают отдельные задачи, но не понимают контекст бизнеса: брендбук, целевую аудиторию, стратегические ограничения.

## Решение

Agent XXL — AI-платформа, где четыре специализированных агента работают как слаженная команда. Каждый агент отвечает за свою область и принимает решения на основе пользовательских знаний (скиллов).

| | Агент | Задача | Инструменты | Метрика |
|---|---|---|---|---|
| 📊 | **Аналитик** | Мониторинг метрик, обнаружение аномалий, прогнозирование | Reports API, статмодели | **ROAS** |
| ✏️ | **Копирайтер** | Генерация текстов объявлений, A/B тестирование | LLM, шаблоны | **CTR** |
| ⚙️ | **Оптимизатор** | Управление ставками, перераспределение бюджетов | Bidding API, правила | **CPA** |
| 🛡️ | **Аудитор** | Проверка качества аккаунта, поиск конфликтов | Чеклисты, QS | **Клики** |

**Ключевое отличие:** агенты — не универсальные боты, а узкие специалисты. Аналитик не пишет тексты. Копирайтер не управляет ставками. Это позволяет давать точные инструкции и измерять эффект каждого отдельно.

## Текущий статус

**Frontend-прототип (MVP)** — интерактивный React-дашборд с имитацией данных и полным UI для всех функций.

## Скиллы — пользовательские знания

Пользователь загружает документы (PDF, MD, TXT), которые определяют поведение агентов. Брендбук компании становится правилами для Копирайтера. Стратегия ставок — инструкциями для Оптимизатора. Чеклист аудита — базой для Аудитора.

### Skill Pipeline

Каждый документ проходит четырёхэтапную обработку:

| Шаг | Этап | Что происходит |
|-----|------|----------------|
| 1 | **Парсинг** | OCR, извлечение текста, таблиц, структуры |
| 2 | **LLM-экстракция** | Типизированные правила: тон, запреты, форматы, аудитория |
| 3 | **Конфликты** | Семантическое сравнение с существующими скиллами |
| 4 | **Версионирование** | Снимок, метрики до/после через 7 дней, откат |

Пользователь видит извлечённые правила, может отредактировать каждое, отключить ненужные и разрешить конфликты перед применением.

---

## Три уровня автономии

Пользователь выбирает степень контроля для каждой кампании. Доверие растёт постепенно — система зарабатывает право на большую самостоятельность.

| Режим | Агенты делают сами | Требует подтверждения |
|-------|--------------------|-----------------------|
| 👁 **Наблюдение** | Ничего — только рекомендации | Любое действие |
| ⚡ **Полуавтомат** | Ставки ±20%, минус-слова, A/B | Новые кампании, бюджет >30% |
| 🤖 **Автопилот** | Всё в рамках лимитов | Выход за границы |

## Управление кампаниями

### Жизненный цикл

Кампания проходит через состояния: **Черновик → На проверке → Активна → Оптимизация → Пауза/Архив**. На каждом этапе работают разные агенты.

### Создание через чат

Вместо формы с 50 полями — пользователь описывает цель: «Хочу лиды на ремонт квартир в СПб до 500₽». Четыре агента параллельно готовят: прогноз (Аналитик), тексты (Копирайтер), ставки (Оптимизатор), чеклист (Аудитор). Пользователь корректирует и запускает.

### Источники кампаний

| Источник | Описание |
|----------|----------|
| **Синхронизация** | Импорт существующих кампаний из Google Ads, далее каждые 15 минут |
| **Wizard (чат)** | Пользователь описывает цель в свободной форме, агенты подготавливают структуру |
| **Предложение агента** | Аналитик видит возможность и предлагает создать кампанию на основе данных |
| **Клонирование** | Копирование существующей кампании на другой гео/бюджет/аудиторию |

## Структура дашборда

### Вкладка «Обзор»
- 4 метрик-карты: клики, показы, конверсии, CPA (с трендами)
- Area-график динамики (клики + конверсии, двойная ось Y)
- Pie-чарт каналов трафика (Поиск, РСЯ, Ретаргетинг, Видео)
- Таблица кампаний: название, статус, бюджет/расход, CPA, ROAS, тренд

### Вкладка «Кампании»
- Раскрываемые карточки с ключевыми словами (QS, CPA) и объявлениями (CTR, A/B статус)
- Лента действий агентов по каждой кампании
- Индикатор уровня автономии

### Вкладка «Агенты»
- 4 summary-карты агентов (статус, impact, кол-во скиллов)
- Bar-чарт эффективности (задачи vs принятые рекомендации)
- Раскрываемые карточки с подвкладками: Скиллы, Версии, Метрики влияния

### Правая панель (AI)
- Рекомендации от агентов с кнопками действий
- Чат с ассистентом (CPA, бюджет, объявления, скиллы)

## Техническая архитектура

### Frontend (реализовано)

| Технология | Назначение |
|------------|------------|
| **React 18 + Vite 5** | Компонентная архитектура, быстрая сборка |
| **Recharts** | Графики: Area, Pie, Bar, Line |
| **Lucide React** | Иконки |
| **CSS-in-JS (inline)** | Стили без внешних зависимостей |

### Backend (спроектировано)

| Компонент | Технология |
|-----------|------------|
| **API** | FastAPI |
| **Оркестрация LLM** | LangGraph |
| **Векторный поиск** | Qdrant / pgvector |
| **База данных** | PostgreSQL |
| **Очереди** | Celery / Temporal |
| **Кеширование** | Redis |
| **Реалтайм** | WebSocket |
| **Google Ads** | Google Ads API v17, двусторонняя синхронизация |

### Дизайн-система

- **Фон:** `#F3EDE5` (тёплый бежевый)
- **Поверхности:** `#FFFFFF` с тенями
- **Акцент:** `#C47B5B` (терракотовый)
- **Цвета агентов:** Google-палитра — синий `#4285F4`, жёлтый `#D09D00`, зелёный `#34A853`, красный `#EA4335`
- **Шрифт:** Inter
